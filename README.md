# TestWab
📑 คู่มือการใช้งานชุดสาธิตระบบควบคุม Digital Twin (Demo System Guide)
คู่มือฉบับนี้จัดทำขึ้นเพื่ออธิบายโครงสร้างระบบซอฟต์แวร์ 4 เลเยอร์ (Three.js Web ➔ Python FSM ➔ Rust Socket Bridge ➔ STM32 Hardware) สำหรับชุดทดลองสาธิตการทำงานควบคุมบอร์ดคอนโทรลเลอร์

🏗️ 1. โครงสร้างสถาปัตยกรรมระบบ (System Architecture)
ระบบถูกออกแบบเป็นลักษณะ Modular ประกอบด้วย 3 โครงสร้างหลักภายใน Repository:

cad/: เลเยอร์หน้าบ้าน (Frontend) แสดงผลโครงสร้างโมเดลเครื่องจักร 3D Twin ทำงานบน Three.js (HTML5/JavaScript)

python_backend/: เลเยอร์สมองกล (FSM Brain Loop) คำนวณตรรกะ 16 สเตป ปล่อยบิต IO Mapping และบันทึกประวัติลงฐานข้อมูล SQLite

rust_bridge/: เลเยอร์สะพานเครือข่ายความเร็วสูง (Network Protocol Bridge) ทำหน้าที่เปิดท่อ Raw Socket สตรีมข้อมูลดิบระหว่าง Python ส่งตรงลงพอร์ตเครือข่ายของบอร์ดจริง

⚙️ 2. สิ่งที่ต้องเตรียมในเครื่องคอมพิวเตอร์ (Prerequisites)
ก่อนเริ่มทำการรันระบบ ให้ติดตั้งเครื่องมือพื้นฐานลงในระบบปฏิบัติการ Windows ดังนี้:

Python (เวอร์ชัน 3.10 ขึ้นไป) พร้อมติดตั้งโมดูลเสริมผ่าน Terminal:

Bash
pip install websockets
Rust Compiler (ผ่านการติดตั้ง rustup.rs) เพื่อใช้งานคำสั่งระบบ cargo

Hardware Accessory: ตัวแปลงสัญญาณ USB-to-LAN Adapter สำหรับคอมพิวเตอร์ที่ไม่มีพอร์ต LAN ออนบอร์ด

🔌 3. การตั้งค่าระบบเครือข่าย (Network Setup - Static IP)
เนื่องจากบอร์ด STM32 ปลายทางถูกระบุพิกัด Network คงที่ไว้ที่เลข 192.168.0.100 พอร์ต 502 คอมพิวเตอร์ที่สั่งรันต้องขยับเข้าไปอยู่ในวงเครือข่ายเดียวกันผ่านขั้นตอนต่อไปนี้:

เสียบตัวแปลง USB-to-LAN เข้าคอมพิวเตอร์ และเสียบสาย LAN เชื่อมเข้าหาบอร์ด Demo ให้แน่น

กดปุ่ม Windows + R พิมพ์ ncpa.cpl แล้วกด Enter

คลิกขวาที่การ์ดแลนใบใหม่ของตัวแปลง USB-to-LAN (เช่น Ethernet 2 หรือ Realtek USB Controller) เลือก Properties

ดับเบิ้ลคลิก Internet Protocol Version 4 (TCP/IPv4)

ติ๊กเลือกช่อง Use the following IP address: แล้วระบุตัวเลข:

IP Address: 192.168.0.22

Subnet Mask: 255.255.255.0

กดปุ่ม OK เพื่อยืนยันการบันทึกค่า

🚀 4. ลำดับสเตปขั้นตอนการสั่งเดินระบบ (Startup Sequence)
⚠️ ข้อควรระวัง: เพื่อป้องกันการชนกันของพอร์ตสื่อสารและลดความหน่วงเน็ตเวิร์ก ต้องเปิดโปรแกรมเรียงตามคิว 1 ➔ 2 ➔ 3 นี้อย่างเคร่งครัด

สเตปที่ 4.1: เปิดสะพานเครือข่ายฝั่งฮาร์ดแวร์ (Rust Layer)
เปิดโปรแกรม VS Code เข้าไปที่โฟลเดอร์หลักของโปรเจกต์

เปิดหน้าต่าง Terminal บานแรกขึ้นมา แล้วใช้คำสั่งย้ายพาร์ทไปยังโฟลเดอร์รัน:

PowerShell
cd TestWab-main
cd rust_bridge
สั่งคอมไพล์และปล่อยรันระบบ:

PowerShell
cargo run
จุดเช็คสถานะ: หน้าจอต้องขึ้นข้อความสีเขียวแจ้งว่า ✅ [Rust -> STM32] Connected to H7 Core Successfully at 192.168.0.100:502

สเตปที่ 4.2: เปิดระบบ FSM สมองกลหลัก (Python Layer)
ที่แถบ Terminal บานเดิม ให้กดคลิกเครื่องหมายบวก + ด้านขวาบน เพื่อเปิดหน้าต่างบานใหม่แยกอิสระ

ใช้คำสั่งย้ายพาร์ทไปยังโฟลเดอร์ประมวลผล Python FSM:

PowerShell
cd TestWab-main
cd python_backend
พิมพ์คำสั่งสั่งเริ่มทำงาน:

PowerShell
python gateway_fsm.py
จุดเช็คสถานะ: หน้าจอต้องขยับพ่นข้อความสีเหลืองตัวหนาว่า 💡 [Python -> Rust] Persistent Connection Established! จากนั้นลอจิก FSM จะเริ่มบันทึกฐานข้อมูล SQLite ทันที

สเตปที่ 4.3: รันหน้าแดชบอร์ดควบคุมสั่งงาน (Web 3D Twin)
เปิดโปรแกรม File Explorer ในคอมพิวเตอร์ เดินทางเข้าไปที่โฟลเดอร์:
...\TestWab-main\TestWab-main\cad

ดับเบิ้ลคลิกเปิดไฟล์หน้าเว็บที่ชื่อว่า index1.html ขึ้นมา (ตัวระบบจะรันผ่าน Google Chrome โดยอัตโนมัติ)

รอให้โมเดลเครื่องจักร 3D Twin ประมวลผลและซิงค์ข้อมูลโครงข่ายสำเร็จ

เอาเมาส์คลิกปุ่มสีเขียว ▶ Start บนหน้าเว็บเพื่อเริ่มต้นการทดลองควบคุมชุดเดโมได้ทันที!

🛠️ 5. วิธีการแก้ไขปัญหาเบื้องต้น (Troubleshooting)
กรณีขึ้นเออร์เรอร์พอร์ตชนกัน [Errno 10048]: เกิดจากมีกระบวนการ Python ค้างสายอยู่ในระบบ ให้พิมพ์คำสั่งนี้ใน Terminal เพื่อตัดการค้างสายก่อนรันใหม่:

PowerShell
taskkill /F /IM python.exe
กรณีไฟเอาต์พุตบนบอร์ดไม่ตอบสนอง: ให้ลองกวาดสายตาตรวจเช็คข้อความใน Terminal ฝั่ง Rust หากขึ้นตัวหนังสือสีแดงแจ้งเตือน ❌ [Heartbeat] STM32 disconnected รัวๆ ให้ทำการถอดและเสียบสาย LAN ที่ตัวแปลงใหม่อีกครั้ง พร้อมกดปุ่ม Reset บนบอร์ดจริง 1 รอบเพื่อกระตุ้นสัญญาณเน็ตเวิร์กครับ
